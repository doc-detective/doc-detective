---
title: Set up your test environment
description: Set up an environment for Doc Detective tests.
---

You can use Doc Detective directly against your production site. For testing, it's often useful to set up a local HTML server to run your tests against. The tutorials that follow fill in a sample intake form, so you need to serve that form locally before you run them.

The sample test specs themselves are hosted online, so you don't need to download them. You pass each one to Doc Detective by URL with the `--input` argument, as shown in the following tutorials.

1. Download the sample intake form, [watson_and_holmes_intake_form.html](https://raw.githubusercontent.com/doc-detective/doc-detective/main/docs/examples/watson-and-holmes/watson_and_holmes_intake_form.html), into an empty folder. The file is self-contained, so it's the only file you need.

   ```shell
   curl -O https://raw.githubusercontent.com/doc-detective/doc-detective/main/docs/examples/watson-and-holmes/watson_and_holmes_intake_form.html
   ```

1. In a terminal, navigate to the folder that contains the form:

   ```shell
   cd <path-to-form-folder>
   ```

   Replace `<path-to-form-folder>` with the actual path to the folder where you saved the form.
1. Install and run a webserver. Running your tests against a local server allows you to make edits and see them in real time. Also, you need to serve the files in a web browser to experiment with video recording and other features.

   Any web server works, but here are a few options (unassociated with Doc Detective):

   <Tabs>
      <Tab language='python' title='Python'>
         To run a local server with Python, you can use the built-in `http.server` module:

         ```bash
         python -m http.server 8080
         ```
      </Tab>
      <Tab language='node' title='Node.js'>
         To run a local server with Node.js, you can use the `http-server` package:

         ```bash
         npx http-server -p 8080
         ```
      </Tab>
      <Tab language='vscode' title='VSCode'>
         {/* vale off */}

         In VSCode, search for and install the [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) extension. It gives you a local development server with live reload capability.

         {/* vale on */}

         <Info>
         To match the Doc Detective tutorials, set the port number to `8080` in your VSCode settings.
         </Info>
      </Tab>
   </Tabs>

   Once the server is started, you can open a browser window to `http://localhost:8080/watson_and_holmes_intake_form.html`. If you see the rendered intake form, you're ready to start testing.
