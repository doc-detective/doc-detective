Downloading and running Doc Detective is straightforward. You can install Doc Detective using NPX (Node.js) or Docker. Follow the instructions for your preferred method:

Choose *NPX* if you have Node.js installed and want to install Doc Detective on your machine locally. Choose *Docker* if you can't install applications locally, prefer containerized environments, need to avoid Node.js installation, or want to isolate Doc Detective from your system.

## Runtime installation

Running `npm i doc-detective` installs the CLI and pre-installs heavy runtime assets—browsers (Chrome, Firefox), drivers (ChromeDriver, Geckodriver), ffmpeg, and the npm packages that drive them (webdriverio, appium, sharp, etc.)—into `<os.tmpdir()>/doc-detective/` (or `DOC_DETECTIVE_CACHE_DIR`). This keeps your install ready to run tests without a separate step.

The pre-install runs in a child process and captures its output, so npm's deprecation warnings from transitive dependencies won't clutter your terminal.

### Opt out of pre-installation

To skip the heavy asset pre-install, set `DOC_DETECTIVE_INSTALL_RUNTIME=0` (also accepts `false`, `no`, or `off`):

```bash
DOC_DETECTIVE_INSTALL_RUNTIME=0 npm i doc-detective
```

With this option, the CLI install stays small. Heavy assets install lazily the first time a test needs them.

### Pre-install manually

If you opted out or need to reinstall, pre-install all runtime assets:

```bash
doc-detective install all --yes
```

### Install specific components

Install only what your tests need:

```bash
doc-detective install browsers chrome
doc-detective install runtime webdriverio appium
doc-detective install agents
```

### Check installation status

See what's installed versus what the current version expects:

```bash
doc-detective install status
```

### Override the cache location

The default cache location (`<os.tmpdir()>/doc-detective/`) may not persist across reboots on some systems. For containers, CI runners, or any environment where you want the cache to survive, set a custom location:

```bash
DOC_DETECTIVE_CACHE_DIR=/opt/doc-detective doc-detective install all --yes
```

You can also set `cacheDir` in your `.doc-detective.json` config file or use the `--cache-dir` CLI flag.

## Auto-update

By default, Doc Detective checks the npm registry on startup and updates itself when a newer version is available:

- **Global installs** (`npm i -g doc-detective`): Runs `npm install -g doc-detective@<latest>` and re-executes.
- **npx invocations**: Re-executes with `npx -y doc-detective@latest`.
- **Local project installs**: Prints an "update available" message instead of modifying your `package.json`.

To disable auto-update, use any of these methods:

- Pass `--no-auto-update` on the CLI
- Set `autoUpdate: false` in your `.doc-detective.json` config file
- Set the `DOC_DETECTIVE_SKIP_AUTO_UPDATE=1` environment variable

CI environments (where `process.env.CI` is set) skip the update check automatically.
