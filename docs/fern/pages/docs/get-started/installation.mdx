You can run Doc Detective directly with npx (no installation required), install it globally with npm, or use Docker.

Choose *npx* to try Doc Detective or run occasional tests without installing anything. Choose *global installation* (`npm i -g doc-detective`) to run `doc-detective` from anywhere for regular use. Choose *Docker* if you prefer isolation or want to avoid Node.js installation.

Install Doc Detective globally for regular use. After installation, you can run `doc-detective` from any directory.

1. Install prerequisites:

    - [Node.js](https://nodejs.org/) v18 or later

2. In a terminal, install Doc Detective globally:

    <CodeBlock language="bash" title="Install Doc Detective">

    </CodeBlock>

  <Tab title="Docker" language="docker">

Run Doc Detective in a container. This is useful if you prefer isolation or want to avoid Node.js installation.

1. Install prerequisites:

    - [Docker](https://docs.docker.com/get-docker/)
    {/* step {"stepId":"check-docker-link","description":"Verify Docker installation link","checkLink":"https://docs.docker.com/get-docker/"} */}

2. Save this file to your current directory as _sample.spec.json_:

    <Code src="sample.spec.json" />

3. Run a test, pointing to the file you just created:

    ```bash
      docker run -v .:/app docdetective/docdetective --input /app/sample.spec.json
    ```
    {/* step {"stepId":"run-docker-test","description":"Run Doc Detective test with Docker","runShell":{"command":"docker run -v .:/app docdetective/docdetective --input /app/sample.spec.json","workingDirectory":".","timeout":180000}} */}

    <Note>

    The `-v` argument mounts your current directory (`.`) to `/app` in the container, giving Doc Detective access to your test specification and enabling it to output results and screenshots. If your test specification is hosted at an HTTPS URL, the `-v` argument is not necessary.

    </Note>

  </Tab>

Congratulations! You've run your first test with Doc Detective. :tada:

You should see output in the terminal indicating the progress and results of the test. Results are also saved in a `testResults-<timestamp>.json` file in the current directory (configurable with the `--output` option), and this test saves a screenshot to *example.png*.
